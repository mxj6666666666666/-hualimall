# 一、前端开发的本质（你要让AI干什么）

你要让AI帮你做的不是“写HTML”，而是：

```
页面结构（HTML + Vue/React）
+ 样式（CSS）
+ 交互（JS）
+ 调后端接口（fetch/axios）
```

👉 所以提示词必须包含这4部分要求

------

# 二、通用提示词模板（核心！！！）

你以后所有页面都用这个模板改一改：

```
请帮我生成一个基于Thymeleaf的前端页面：

页面名称：XXX页面（例如：首页/商品列表/购物车）

技术要求：
1. 使用原生HTML + CSS + JavaScript（不要Vue/React）
2. 页面风格简洁，类似电商网站
3. 使用axios调用后端接口
4. 后端返回格式为：
   { code: 200, message: "success", data: ... }

功能要求：
[写具体功能]

接口信息：
[写接口路径]

输出要求：
1. 给出完整HTML文件（可直接运行）
2. 包含CSS样式
3. 包含JS逻辑
4. 代码要有注释
```

# 三、你这个项目必须做的5个页面（+提示词）

## 1️⃣ 登录页（第一个做）

### 提示词：

```
请帮我生成一个基于react的登录页面：

技术要求：
1. 使用HTML + CSS + JavaScript
2. 风格简洁，居中登录框
3. 不使用前端框架

功能：
1. 输入用户名和密码
2. 点击登录按钮调用接口
3. 登录成功后保存token到localStorage
4. 跳转到商品列表页

接口：
POST /api/user/login

请求参数：
{
  "username": "",
  "password": ""
}

返回：
{
  code: 200,
  data: {
    token: "xxx"
  }
}

要求：
1. 使用fetch发送请求
2. 代码完整可运行
3. 有基础样式（类似京东登录）
```

------

## 2️⃣ 商品列表页（核心页面）

### 提示词：

```
请帮我生成一个商品列表页面（Thymeleaf）：

功能：
1. 页面加载时请求商品列表
2. 展示商品（图片、名称、价格）
3. 点击商品跳转详情页
4. 每个商品有“加入购物车”按钮

接口：
GET /api/product/list

返回：
{
  code: 200,
  data: [
    {
      id: 1,
      name: "商品A",
      price: 99.00,
      imageUrl: "xxx"
    }
  ]
}

要求：
1. 使用fetch获取数据
2. 使用grid布局展示商品
3. 页面美观（卡片式）
4. 点击加入购物车调用接口：
   POST /api/cart/add
5. 请求头携带token：
   Authorization: Bearer xxx
```

# 四、一个非常关键的点（很多人会犯错）

👉 你必须额外告诉AI：

```
所有接口请求必须带token：

headers: {
  "Authorization": "Bearer " + localStorage.getItem("token")
}
```

否则你后端JWT会全部401 ❌

# 五、进阶提示词（让页面更像“真实项目”）

你可以在提示词最后加一句：

```
请让页面更接近真实电商网站（类似京东/淘宝的简化版），
包括：
- 顶部导航栏（登录信息）
- 简单响应式布局
- 基础交互提示（alert或提示信息）
```

👉 这一步 = 面试加分点 💥

# 六、开发顺序（非常重要）

你不要乱做，按这个顺序：

```
1. 登录页
2. 商品列表页
3. 商品详情页
4. 购物车
5. 下单
```

👉 每做完一个页面就**马上调接口测试**

# 七、我给你一个“最强技巧”（AI效率翻倍）

当AI生成页面后，你再继续问：

```

1. 代码结构更清晰
```

👉 这一步能让你的项目**从学生作业 → 接近真实项目**
