## 4.3 Controller（`/payments`）

```java
package com.xinjiema.hualimall.controll;

import com.xinjiema.hualimall.pojo.CreatePaymentRequest;
import com.xinjiema.hualimall.pojo.Payment;
import com.xinjiema.hualimall.pojo.Result;
import com.xinjiema.hualimall.service.PaymentService;
import lombok.extern.slf4j.Slf4j;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.web.bind.annotation.*;

@Slf4j
@RestController
@RequestMapping("/payments")
public class PaymentController {

    @Autowired
    private PaymentService paymentService;

    @PostMapping
    public Result<Payment> createPayment(@RequestBody CreatePaymentRequest req) {
        log.info("创建支付单: {}", req);
        return Result.success(paymentService.createPayment(req));
    }

    @GetMapping("/{id}")
    public Result<Payment> getPaymentById(@PathVariable Long id) {
        return Result.success(paymentService.getById(id));
    }

    @GetMapping("/order/{orderId}")
    public Result<Payment> getPaymentByOrderId(@PathVariable Long orderId) {
        return Result.success(paymentService.getLatestByOrderId(orderId));
    }

    @PutMapping("/{id}/close")
    public Result<String> closePayment(@PathVariable Long id) {
        paymentService.closePayment(id);
        return Result.success("关闭成功");
    }

    // 回调接口
    @PostMapping("/notify/alipay")
    public String notifyAlipay(@RequestParam java.util.Map<String, String> params) {
        return paymentService.handleAlipayNotify(params);
    }

    @PostMapping("/notify/wechat")
    public String notifyWechat(@RequestBody String body) {
        return paymentService.handleWechatNotify(body);
    }
}
```

## 4.5 Service 实现核心规则（你必须落地）

在 `PaymentServiceImpl` 里按下面规则写：

1. **创建幂等**：先查该订单 `status in (PENDING, PROCESSING)`，有则直接返回。  
2. **订单合法性**：订单存在且 `status=0` 才允许创建支付单。  
3. **金额以订单为准**：`payment.amount = orders.total_amount`。  
4. **支付状态终态保护**：`SUCCESS/CLOSED/FAILED` 为终态，禁止回滚。  
5. **关闭规则**：仅 `PENDING/PROCESSING` 可关闭。`SUCCESS` 关闭直接报错。  
6. **回调幂等**：若支付单已 `SUCCESS`，重复成功回调直接返回成功字符串。  
7. **金额校验**：回调金额不一致直接拒绝并记录日志。  
8. **订单联动**：支付成功更新 `orders.status=1`；关闭不应把已支付订单改回未支付。

## 5. Mapper 接口建议

```java
package com.xinjiema.hualimall.mapper;

import com.xinjiema.hualimall.pojo.Payment;
import org.apache.ibatis.annotations.Mapper;
import org.apache.ibatis.annotations.Param;

import java.time.OffsetDateTime;
import java.util.List;

@Mapper
public interface PaymentMapper {
    void insert(Payment payment);
    Payment selectById(@Param("id") Long id);
    Payment selectLatestByOrderId(@Param("orderId") Long orderId);
    Payment selectActiveByOrderId(@Param("orderId") Long orderId); // PENDING/PROCESSING

    int updateStatusToClosed(@Param("id") Long id, @Param("closeTime") OffsetDateTime closeTime);
    int updateStatusToSuccess(@Param("id") Long id, @Param("channelTradeNo") String channelTradeNo, @Param("paidTime") OffsetDateTime paidTime);
    int updateStatusToFailed(@Param("id") Long id, @Param("failReason") String failReason);

    List<Payment> selectTimeoutPending(@Param("now") OffsetDateTime now);
}
```

> 注意：`@Param("xxx")` 名称必须与 XML 里的 `#{xxx}` 一致（你当前项目就是这套规则）。

## 6. 与现有 `AlipayController` 的处理建议

你现在项目里已经有 `controll\AlipayController.java`，且也是 `@RequestMapping("/payments")`。  
要避免路由冲突，建议二选一：

1. **推荐**：把其支付逻辑下沉到 `PaymentServiceImpl`，保留一个统一 `PaymentController`。  
2. 或把旧控制器改为 `/alipay` 前缀，仅保留第三方能力，不直接作为前端支付主接口。

## 7. 开发顺序（按联调优先级）

1. 先完成 `POST /payments`（返回 `payUrl/codeUrl`）。  
2. 再完成 `GET /payments/{id}`（让前端轮询）。  
3. 再做回调 `/payments/notify/alipay|wechat`（成功后订单 `0 -> 1`）。  
4. 再做 `PUT /payments/{id}/close`。  
5. 最后做 `GET /payments/order/{orderId}`（刷新恢复支付单）。

## 9. 你写代码时最容易踩坑的 6 件事

1. `PaymentMapper.java` 和 `PaymentMapper.xml` 方法名不一致。  
2. XML 里 `collection` 名和 `@Param` 名不一致。  
3. 回调验签失败还返回 success（这会被攻击）。  
4. 已 `SUCCESS` 后又被 `CLOSED/FAILED` 覆盖。  
5. 回调未校验金额、商户号、订单号映射关系。  
6. 创建支付单没有做幂等，导致同订单多个进行中支付单。
