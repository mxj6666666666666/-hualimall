### 核心分层职责说明（与你图中对应）

| 层             | 包名               | 职责                                                         |
| -------------- | ------------------ | ------------------------------------------------------------ |
| **Controller** | `controller`       | 接收前端 JSON 请求，参数校验，调用 Service，返回 `Result` 统一格式。 |
| **Service**    | `service` + `impl` | 业务逻辑处理，事务管理，调用 Mapper 和 Cache。               |
| **Cache**      | `cache`            | 封装 Redis 操作，供 Service 调用，实现缓存穿透、预热、更新策略。 |
| **Mapper**     | `mapper`           | MyBatis-Plus BaseMapper 继承，数据库 CRUD 操作。             |
| **Entity**     | `entity`           | 与数据库表字段一一对应，使用 `@TableName`、`@TableId` 等注解。 |
| **DTO**        | `dto`              | 前端交互对象，隔离数据库表结构变化，避免直接暴露 Entity。    |

### 前后端分离特别说明

- **删除 Thymeleaf 依赖**：`pom.xml` 中移除 `spring-boot-starter-thymeleaf`，`resources/templates` 目录可删除。
- **跨域处理**：在 `WebMvcConfig` 中配置全局 CORS 或使用 `@CrossOrigin` 注解。
- **统一响应格式**：`Result` 类封装 `code`、`message`、`data`，前端根据状态码处理。
- **认证方案**：通常使用 JWT 放在 `Authorization` 头中，由拦截器校验。

### 可选扩展模块目录

```
├── security/           # Spring Security 配置（若引入权限框架）
├── mq/                 # 消息队列生产者/消费者（如 RabbitMQ 处理订单异步）
├── job/                # 定时任务（如 XXL-Job 或 @Scheduled）
├── third/              # 第三方服务对接（支付宝支付、短信等）
└── listener/           # 事件监听器（如订单完成后发积分）
```

### 关键配置文件示例 `application.yml`

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/mall?useSSL=false&serverTimezone=Asia/Shanghai
    username: root
    password: 123456
  redis:
    host: localhost
    port: 6379
    database: 0
  jackson:
    date-format: yyyy-MM-dd HH:mm:ss
    time-zone: GMT+8

mybatis-plus:
  mapper-locations: classpath:mapper/*.xml
  global-config:
    db-config:
      id-type: auto
      logic-delete-field: deleted
  configuration:
    log-impl: org.apache.ibatis.logging.stdout.StdOutImpl

server:
  port: 8080
```

按照以上结构组织代码，既符合 Spring Boot 分层规范，又能清晰支持前后端分离的商城业务扩展。
