# 页面讲解（认证与商品）

## src/views/auth/LoginView.vue（登录页）
核心流程：
1. 表单校验：用户名、密码不能为空
2. `userApi.login(form)` 获取 token
3. `authStore.setToken(...)`
4. `userApi.profile()` 获取用户信息
5. `authStore.setUser(...)`
6. 跳转到 `redirect` 或 `/products`

关键点：
- `@submit.prevent`：阻止浏览器默认提交行为
- `v-model.trim`：输入自动去掉首尾空格
- `loading/error`：控制按钮状态和错误提示

## src/views/auth/RegisterView.vue（注册页）
核心流程：
1. 基础校验
2. `userApi.register(form)`
3. 成功后清空表单并提示“去登录”

## src/views/product/ProductListView.vue（商品列表）
关键能力：
- 条件筛选：关键词、分类、状态
- 分页：`page/pageSize/total`
- 加购物车：未登录先跳登录

核心语法：
- `reactive(query)`：管理查询参数对象
- `computed(totalPages)`：根据总数自动计算总页数
- `onMounted(fetchProducts)`：页面首次加载自动请求

关键代码思路：
```js
const params = {
  keyword: query.keyword || undefined,
  status: query.status === '' ? undefined : query.status,
}
```
作用：把空值转成 `undefined`，避免把“空筛选条件”发给后端。

## src/views/product/ProductDetailView.vue（商品详情）
关键能力：
- 根据路由参数 `:id` 拉详情
- 选择数量加入购物车
- 未登录时跳登录并保留回跳地址

关键点：
- `useRoute()` 读取 `route.params.id`
- `v-model.number` 把输入值转数字
- 基础业务校验：数量必须 > 0
