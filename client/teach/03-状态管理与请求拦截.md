# 状态管理（Pinia）与请求拦截（Axios）

## src/store/modules/auth.js（登录状态中心）
这个文件管理：
- `token`
- 当前用户信息 `user`

### 核心设计
1. **state**：数据源
2. **getters**：派生状态（如 `isLoggedIn`、`role`、`nickname`）
3. **actions**：修改状态（`setToken`、`setUser`、`clearAuth`）

### 关键点
- token 和 user 同步到 `localStorage`，刷新页面后仍可恢复
- `clearAuth()` 会清除内存和本地存储，彻底退出登录

## src/api/request.js（全局请求实例）
它是全项目 HTTP 请求“总入口”。

### 1) 创建 axios 实例
```js
const request = axios.create({
  baseURL: import.meta.env.VITE_API_BASE_URL || '/api',
  timeout: 10000,
})
```

### 2) 请求拦截器（自动带 token）
```js
if (authStore.token) {
  config.headers.Authorization = `Bearer ${authStore.token}`
}
```
含义：登录后每个请求自动带身份令牌。

### 3) 响应拦截器（统一成功失败格式）
```js
if (payload?.code === 200) return payload.data
return Promise.reject(new Error(payload?.message || '请求失败'))
```
含义：后端若约定 `code=200` 才算成功，则只返回 `data`。

### 4) 401 处理
```js
if (error.response?.status === 401) {
  authStore.clearAuth()
  router.push('/login')
}
```
含义：登录过期自动清理并跳登录。

---

## 你要记住的一句话
页面里直接调用 `api` 模块；`api` 模块底层都走 `request.js`，所以 token、错误、401 都是统一管理的。
