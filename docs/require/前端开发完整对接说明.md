# 前端开发完整对接说明（hualimalll）

> 本文用于前后端分离联调，覆盖前端页面所需的核心内容：接口清单、数据结构、状态定义、页面流程、鉴权规则、错误处理与联调约定。

## 1. 项目与接口基础信息

- 后端技术：Spring Boot + MyBatis + PageHelper
- 接口风格：RESTful 为主，同时兼容旧接口路径
- 统一响应结构：`Result<T>`

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

`code` 约定：

- `200`：成功
- `400`：参数/业务校验失败
- `401`：未登录或 token 非法
- `500`：其他运行时错误

---

## 2. 鉴权与请求规范（前端必须）

## 2.1 Token 使用

登录成功后会返回：

```json
{ "code": 200, "message": "success", "data": { "token": "xxx" } }
```

后续受保护接口必须带：

`Authorization: Bearer <token>`

建议前端统一在请求拦截器里注入 token。

## 2.2 需要登录的接口范围

后端拦截范围：

- `/orders/**`、`/order/**`
- `/carts/**`、`/cart/**`
- `/admin/**`
- `/users/me`、`/users/info`、`/user/me`、`/user/info`

> 注意：当前 `/admin/**` 仅做“已登录”校验，未做细粒度角色鉴权。

## 2.3 前端错误处理建议

- 收到 `401`：清空本地登录态，跳转登录页
- 收到 `400`：弹出后端 `message`
- 收到 `500`：统一提示“系统繁忙，请稍后重试”，并记录日志

## 3. 数据结构（前端 TypeScript 可直接使用）

```ts
type Result<T> = {
  code: number;
  message: string;
  data: T;
};

type PageResult<T> = {
  total: number;
  rows: T[];
};

type User = {
  id: number;
  username: string;
  password?: string;
  nickname: string;
  role: string; // USER / ADMIN(若后续扩展)
  status: number; // 1启用 0禁用
  createTime: string;
  updateTime: string;
};

type Product = {
  id: number;
  name: string;
  categoryId?: number;
  price: string; // BigDecimal 建议按字符串处理
  stock: number;
  imageUrl?: string;
  description?: string;
  status: number; // 1上架 0下架
  createTime: string;
  updateTime: string;
};

type CartItem = {
  id: number;
  userId: number;
  productId: number;
  quantity: number;
  selected: number; // 1选中 0未选
  productName?: string;
  productPrice?: string;
  imageUrl?: string;
  createTime: string;
  updateTime: string;
};

type OrderItem = {
  id: number;
  orderId: number;
  productId: number;
  productName: string;
  productPrice: string;
  quantity: number;
  totalPrice: string;
};

type Order = {
  id: number;
  orderNo: string;
  userId: number;
  totalAmount: string;
  status: number; // 0待支付 1已支付 2已取消 3已完成
  items: OrderItem[];
  createTime: string;
  updateTime: string;
};
```

## 5. 接口清单（按页面可直接对接）

## 5.1 用户与登录

### 5.1.1 注册

- `POST /users`（兼容：`POST /user/register`）
- Body:

```json
{
  "username": "alice",
  "password": "123456",
  "nickname": "Alice"
}
```

- 返回：`Result<string>`

### 5.1.2 登录

- `POST /users/sessions`（兼容：`POST /user/login`）
- Body:

```json
{
  "username": "alice",
  "password": "123456"
}
```

- 返回：`Result<{ token: string }>`

### 5.1.3 当前用户信息

- `GET /users/me`（兼容：`/users/info` `/user/me` `/user/info`）
- Header: `Authorization: Bearer <token>`
- 返回：`Result<User>`

---

## 5.2 商品

### 5.2.1 商品列表

- `GET /products`（兼容：`GET /products/list`）
- Query:
  - `page`（默认 1）
  - `pageSize`（默认 10）
  - `categoryId`（可选）
  - `status`（可选）
  - `keyword`（可选）
- 返回：`Result<PageResult<Product>>`

### 5.2.2 商品详情

- `GET /products/{id}`
- 返回：`Result<Product>`

---

## 5.3 购物车（需登录）

### 5.3.1 购物车列表

- `GET /carts`（兼容：`GET /cart/list`）
- 返回：`Result<CartItem[]>`

### 5.3.2 添加购物车

- `POST /carts`（兼容：`POST /cart/add`）
- Body:

```json
{
  "productId": 10001,
  "quantity": 2
}
```

- 返回：`Result<string>`

### 5.3.3 更新购物车项

- `PUT /carts/{id}`
- 兼容旧接口：`PUT /cart/update?id={id}`
- Body（至少传一个字段）:

```json
{
  "quantity": 3,
  "selected": 1
}
```

- 返回：`Result<string>`

### 5.3.4 删除购物车项

- `DELETE /carts/{id}`（兼容：`DELETE /cart/remove/{id}`）
- 返回：`Result<string>`

---

## 5.4 订单（需登录）

### 5.4.1 创建订单

- `POST /orders`（兼容：`POST /order/create`）
- Body 两种方式：

1) 传 `items` 明细下单

```json
{
  "items": [
    { "productId": 10001, "quantity": 2 },
    { "productId": 10002, "quantity": 1 }
  ]
}
```

2) 不传 `items`，后端自动读取“当前用户已勾选购物车项”

- 返回：`Result<Order>`

### 5.4.2 订单分页列表

- `GET /orders`（兼容：`GET /order/list`）
- Query:
  - `page`（默认 1）
  - `pageSize`（默认 10）
- 返回：`Result<PageResult<Order>>`

### 5.4.3 订单详情

- `GET /orders/{id}`
- 返回：`Result<Order>`

### 5.4.4 取消订单

- `PUT /orders/{id}/cancel`（兼容：`PUT /order/cancel/{id}`）
- 返回：`Result<string>`

## 5.5 管理端接口（需登录）

## 5.5.1 商品管理

- `POST /admin/products`：新增商品
- `POST /admin/products/batch`：批量新增
- `PUT /admin/products/{id}`：修改商品
- `PUT /admin/products/batch`：批量修改
- `DELETE /admin/products/{id}`：删除商品
- `DELETE /admin/products/batch`：批量删除

批量示例（新增）：

```json
[
  {
    "name": "测试商品A",
    "categoryId": 1,
    "price": 99.99,
    "stock": 100,
    "status": 1
  },
  {
    "name": "测试商品B",
    "categoryId": 1,
    "price": 59.9,
    "stock": 50,
    "status": 1
  }
]
```

## 5.5.2 订单状态管理

- `PUT /admin/orders/{id}/status`
- 兼容：`PUT /admin/order/{id}/status/update`
- Body:

```json
{
  "status": 1
}
```

状态值：

- `0` 待支付
- `1` 已支付
- `2` 已取消
- `3` 已完成

## 7. 联调与工程约定（推荐）

## 7.1 环境变量

建议前端配置：

- `VITE_API_BASE_URL=http://localhost:8080`（按实际后端端口修改）

## 7.2 请求工具层

建议封装：

1. 请求拦截器：自动加 token
2. 响应拦截器：统一按 `code` 分流
3. 通用分页参数与分页返回类型
4. BigDecimal 金额显示统一格式化（保留两位）

## 7.3 兼容路径策略

新页面优先使用 RESTful 路径（如 `/users`、`/orders`、`/carts`），旧路径仅作为兼容兜底，不建议新代码长期依赖。

---

## 8. 当前已知注意事项（前端需感知）

1. 管理端目前仅“登录可访问”，未在后端强校验 `role=ADMIN`。前端应先做菜单与入口控制，但不要将其视为真正权限边界。  
2. 订单列表/详情当前后端未看到“按当前用户严格过滤”的强约束代码，前端需避免暴露可枚举 id 的高风险操作。  
3. 目前后端未提供专门“支付”接口，订单状态 `1`（已支付）主要通过管理端更新接口变更。  

---

## 9. 前端交付最小闭环清单（建议优先级）

1. 登录注册 + token 持久化  
2. 商品列表/详情 + 加购  
3. 购物车增删改查 + 勾选  
4. 下单 + 订单列表 + 订单详情 + 取消  
5. 管理端商品 CRUD + 批量操作  
6. 管理端订单状态更新  

完成以上即具备可演示的完整商城链路。
