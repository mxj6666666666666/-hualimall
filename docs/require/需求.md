## 一、外包公司眼中的“合格练手项目”标准

| 维度           | 最低要求                                      | 加分项                                          |
| -------------- | --------------------------------------------- | ----------------------------------------------- |
| **功能完整度** | 商品浏览+购物车+下单+管理后台                 | 用户登录注册、订单状态跟踪、简单的库存管理      |
| **技术栈**     | Spring Boot + MyBatis + MySQL                 | Redis（缓存）、JWT（登录）、Swagger（接口文档） |
| **代码规范**   | 有分层（Controller/Service/Mapper）、命名清晰 | 有注释、有异常处理、有日志                      |
| **可运行性**   | 提供SQL脚本、README写清楚启动步骤             | 提供Docker一键启动、有在线演示地址              |
| **真实性**     | 数据不是test1/test2，有像样的商品图片和描述   | 有后台管理界面、有简单的权限区分（用户/管理员） |

**核心原则**：让看的人觉得“这项目拿过来改改就能用”，而不是“这是个课堂作业”。

## 二、推荐的功能范围（不大不小，2-3周能做完）

不要做淘宝那种完整商城，外包公司也不需要。做**最小可用但完整**的版本：

### 用户端（5-6个页面）
- 首页：商品列表（带分类/搜索）
- 商品详情页：展示图片、价格、描述
- 购物车页：增删改查商品
- 下单页：填写地址、提交订单
- 订单列表页：查看历史订单
- 个人中心：修改密码、收货地址管理

### 管理端（3-4个页面）
- 商品管理：增删改查、上下架
- 订单管理：查看订单、修改状态（待发货→已发货→已完成）
- 简单的数据统计（可选）：今日订单数、销售额

### 后端接口（15-20个）
- 用户：登录、注册、获取个人信息
- 商品：列表、详情、分类查询
- 购物车：添加、删除、更新数量、查询
- 订单：创建、列表、详情、取消
- 管理端：商品CRUD、订单列表、订单状态修改

**明确不做**（节省时间，也避免复杂风险）：
- ❌ 真实支付（用“模拟支付”或“货到付款”代替）
- ❌ 优惠券/拼团/秒杀
- ❌ 微信登录/支付宝支付
- ❌ 复杂的权限管理（管理员就一个固定账号）

## 三、技术选型建议（既主流又不复杂）

| 层级         | 技术                  | 原因                                |
| ------------ | --------------------- | ----------------------------------- |
| 后端框架     | Spring Boot 2.7.x     | 稳定、资料多、AI生成代码成功率高    |
| ORM          | MyBatis-Plus          | 比原生MyBatis简单，AI很擅长生成     |
| 数据库       | MySQL 8.0             | 主流，提供init.sql即可              |
| 缓存（可选） | Redis                 | 做购物车或登录token，加分项         |
| 权限         | JWT（jjwt）           | 无状态、简单，不用学Spring Security |
| 接口文档     | Swagger/Knife4j       | 自动生成，显得专业                  |
| 前端（可选） | Thymeleaf + Bootstrap | 不用单独写Vue，减少工作量           |
| 项目管理     | Maven                 | 标准                                |

**如果用AI生成**，建议先让AI生成Spring Boot + MyBatis-Plus + Thymeleaf的脚手架，然后逐个模块让AI补充。

## 五、让项目“看起来很专业”的5个细节

这些细节花不了多少时间，但外包公司看了会觉得你靠谱：

### 1. README写得好（最重要）
```markdown
# 项目名称：简单商城系统

## 技术栈
Spring Boot 2.7 + MyBatis-Plus + MySQL + JWT + Swagger

## 快速启动
1. 导入SQL脚本（/docs/schema.sql）
2. 修改application.yml中的数据库密码
3. 运行Application.java
4. 访问 http://localhost:8080

## 测试账号
- 普通用户：user/123456
- 管理员：admin/123456

## 接口文档
启动后访问 http://localhost:8080/swagger-ui.html

## 演示地址
http://xxx.xxx.xxx（如果有部署）
```

### 2. 提供SQL脚本和数据
- 至少插入10条商品数据，图片用网络图片URL
- 商品名称不要是“商品1”“商品2”，要像“华为Mate60 Pro”“iPhone 15”
- 价格、库存要有真实感

### 3. 统一返回格式
```java
// 所有接口返回统一格式
{
  "code": 200,
  "message": "成功",
  "data": {}
}
```

### 4. 异常处理
- 库存不足时返回明确提示：“库存不足，剩余3件”
- 未登录时返回：“请先登录”
- 不要直接返回500错误页面

### 5. 项目结构清晰
```
src/main/java/com/demo/mall
├── controller/     // 接口层
├── service/        // 业务层
├── mapper/         // 数据层
├── entity/         // 实体类
├── dto/            // 数据传输对象
├── config/         // 配置类（JWT、Swagger）
└── common/         // 通用类（返回结果、异常）
```

## 七、一个更快的思路：用AI生成+手动调整

你现在可以用AI，完全可以**3倍速**完成这个项目：

### 具体操作
1. 让AI生成项目结构：*“用Spring Boot生成一个商城项目的目录结构，包含用户、商品、购物车、订单四个模块”*
2. 让AI逐张表生成：*“生成用户表的实体类、Mapper、Service、Controller，使用MyBatis-Plus”*
3. 让AI生成接口：*“生成商品列表接口，支持分页和按分类筛选”*
4. 自己调整：把AI生成的代码中不合适的地方改掉（比如返回格式统一）

### 注意
- **不要完全无脑复制**：每一段AI生成的代码，至少看懂80%
- **关键逻辑手写**：下单时的库存扣减、购物车的并发问题，自己写并理解
- **测试自己做**：用Postman把所有接口测一遍
